# YOLOv8 Production Studio

A real-time object detection web application built with YOLOv8, OpenCV, and Streamlit. It streams your webcam feed directly in the browser, draws bounding boxes around detected objects, and logs what it sees — all in real time.

Live demo: [object-class-detection-model.streamlit.app](https://detection-model.streamlit.app/)

---

## What This Project Does

You point a webcam at something. The app figures out what it is looking at, draws a box around it, labels it with a confidence percentage, and shows you a live log of everything it detects. If your machine has a CUDA-capable GPU, it uses that automatically. If not, it falls back to the CPU without any fuss.

The project is split into two parts that work together:

- `yolo-webcam.py` — runs locally on your machine, opens your webcam, lets you test detection live, and when you press `q` to quit, it saves the model weights and a configuration file to disk.
- `app.py` — the Streamlit web app that loads those saved files and runs detection through your browser using WebRTC.

---

## Project Structure

```
.
├── app.py                  # Streamlit web application
├── yolo-webcam.py          # Local webcam script (run this first)
├── requirements.txt        # Python dependencies
├── model_config.pkl        # Generated by yolo-webcam.py (class names, metadata)
└── app_model_weights.pt    # Generated by yolo-webcam.py (saved YOLO model)
```

The two generated files (`model_config.pkl` and `app_model_weights.pt`) do not exist until you run `yolo-webcam.py` and press `q` to exit. The Streamlit app will not start without them.

## Getting Started

### Step 1 — Prerequisites

You need Python 3.9 or higher and pip installed. A webcam is required for the local script. For the hosted Streamlit app, you just need a browser with webcam access.

If you want GPU acceleration, make sure you have:

- An NVIDIA GPU
- CUDA Toolkit installed (version compatible with your PyTorch build)
- The correct version of `torch` with CUDA support (see the PyTorch installation guide at pytorch.org/get-started)

### Step 2 — Clone and Install

```bash
git clone https://github.com/your-username/your-repo-name.git
cd your-repo-name
pip install -r requirements.txt
```

If you are using a virtual environment, activate it before running pip.

### Step 3 — Run the Local Webcam Script

This step generates the two files the Streamlit app depends on.

```bash
python yolo-webcam.py
```

A window will open showing your webcam feed with detection boxes drawn in real time. When you are satisfied, press `q` to close the window. The script will then save `app_model_weights.pt` and `model_config.pkl` in the same directory.

You should see output like this in your terminal:

```
Webcam window closed. Starting pipeline serialization...
Saved model weights structure to: 'app_model_weights.pt'
Successfully generated PICKLE file configuration: 'model_config.pkl'
```

### Step 4 — Launch the Streamlit App

```bash
streamlit run app.py
```

Open your browser to the URL shown in the terminal (usually `http://localhost:8501`). Click Start, allow webcam access when prompted, and detection will begin.

## How the Two Scripts Work Together

### yolo-webcam.py

This script is your local testing environment. It:

- Opens your webcam at 1280x720 resolution
- Loads the base `yolov8n.pt` weights (downloaded automatically on first run)
- Mirrors the camera feed horizontally so the image feels natural
- Draws bounding boxes using `cvzone.cornerRect`
- Labels each detection with its class name and confidence score
- On exit (press `q`), serializes the model weights and metadata to disk so the web app can load them

The script uses the COCO dataset classes by default, which covers 80 common object types including people, vehicles, animals, and everyday items.

### app.py

This is the production-facing web application. It:

- Detects whether a CUDA-capable GPU is available and routes computation accordingly
- Loads the saved model weights and pickle file at startup
- Streams your webcam through WebRTC directly in the browser (no additional software needed)
- Runs YOLOv8 inference on each video frame in real time
- Displays confidence-filtered bounding boxes and labels overlaid on the stream
- Shows a live detection log below the video feed with badge-style labels

The confidence threshold can be adjusted at any time using the sidebar slider without restarting the app.

## Configuration

All runtime settings are controlled from the sidebar inside the Streamlit app.

| Setting | Description | Default |
|---|---|---|
| Detection Confidence Threshold | Minimum confidence score for a detection to appear | 0.45 |
| Processing Engine | Auto-detected (CUDA or CPU) | Auto |
| Loaded Profile | Shows the model version from the pickle file | YOLOv8n-Production |

The "See Pickled Target Classes" expander in the sidebar shows every class name stored in `model_config.pkl`.

## How Object Detection Works (Plain Language)

When a video frame arrives from your webcam, it goes through the following steps:

1. The frame is decoded from WebRTC format into a NumPy array.
2. It is flipped horizontally so left and right match what you see in a mirror.
3. The frame is converted from RGB to BGR, which is what OpenCV and YOLO expect.
4. YOLOv8 runs inference on the frame using the loaded model weights.
5. For each detected object that meets the confidence threshold, the app reads the bounding box coordinates, the class index, and the confidence score.
6. A styled corner rectangle is drawn around the object, and a label is placed above it showing the class name and confidence percentage.
7. The annotated frame is returned and displayed in the browser.
8. The detection labels are also pushed to the log queue, where they appear below the video feed as styled badges.

## Running on a Remote Server or Streamlit Cloud

When deploying to Streamlit Cloud or any remote server:

- GPU acceleration will not be available unless the server has a CUDA-capable GPU configured. The app will automatically fall back to CPU.
- You must upload both `model_config.pkl` and `app_model_weights.pt` alongside `app.py`. The app will not start if either file is missing.
- WebRTC requires a working STUN server connection. The app includes Google's public STUN servers by default, which work in most network environments.
- Browser webcam access over HTTP is blocked by most browsers for security reasons. Streamlit Cloud serves over HTTPS, so this is handled automatically there. If you are self-hosting, make sure you are serving the app over HTTPS.

## Common Issues

**The app shows "Critical Files Missing" and stops**

You need to run `yolo-webcam.py` first and press `q` to generate `model_config.pkl` and `app_model_weights.pt`. Then upload or move those files to the same directory as `app.py`.

**The webcam does not start in the browser**

Check that your browser has granted webcam permission to the site. In Chrome, look for the camera icon in the address bar. On Streamlit Cloud, permissions are requested automatically when you click Start.

**Detection is slow**

If you are on CPU, inference time depends heavily on your machine. YOLOv8n (nano) is the fastest variant. Consider reducing the video resolution in `yolo-webcam.py` if you need more speed, by changing the `cap.set(3, ...)` and `cap.set(4, ...)` values.

**Bounding boxes are offset or misaligned**

The mirroring step in both scripts is intentional and synchronized. The webcam frame is flipped before YOLO sees it, so bounding box coordinates are already in the correct position for display. Do not add additional flip operations.

**No detections appear even though objects are clearly visible**

Try lowering the confidence threshold slider in the sidebar. The default is 0.45, which means the model must be at least 45% confident before it draws a box. Dropping this to 0.25 or 0.30 will show more detections, including less certain ones.

## Extending the Project

**Using a custom-trained model**

Replace the YOLO initialization in `yolo-webcam.py` with a path to your own `.pt` file:

```python
model = YOLO("path/to/your/custom_model.pt")
```

Run the script to generate new `model_config.pkl` and `app_model_weights.pt` files reflecting your custom classes.

**Changing the base model size**

YOLOv8 comes in five sizes: nano (`yolov8n`), small (`yolov8s`), medium (`yolov8m`), large (`yolov8l`), and extra-large (`yolov8x`). Larger models are more accurate but slower. Swap the model name in `yolo-webcam.py`:

```python
model = YOLO("../yolo-weights/yolov8s.pt")
```

**Adjusting webcam resolution**

In `yolo-webcam.py`, change the resolution values:

```python
cap.set(3, 1920)  # Width
cap.set(4, 1080)  # Height
```

Note that higher resolutions will slow down inference if you are on CPU.
